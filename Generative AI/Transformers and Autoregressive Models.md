For each batch, maximize cosine similarity between matched (image, caption) pairs: minimize it for all other pairings
Loss function: InfoNCE Contrastive loss
Intuitively, each image $v_i$ must identify its own caption $t_i$ out of $N$ distractors, and vice-versa

CLIP LOSS
![[Screenshot 2026-03-27 at 6.48.17 PM.png]]
- Numerator : rewards high similarity for the correct pair ($v_i, t_i$) Temperature controls how sharp the distribution is
- Denominator : sums over all N texts in the batch, including N-1 negatives
Converts a probability into a loss: perfect match --> probability 1 --> loss 0
Averages over all anchors in the batch
Ensures both encoders receive gradients, neither encoders is treated as "fixed"

 CLIP was initially used for zero-shot classification
 ![[Screenshot 2026-03-28 at 12.05.11 AM.png|500]]
--> don't need any examples for the classifier, but rely on the training of CLIP to classify the text and image 

CLIP is important for us because:
- the text encoder is the conditioning backbone inside Stable Diffusion (converts prompts into the vectors that guide image generation)
- Compare an image embedding against text embeddings of class labels --> no fine-tuning needed
- CLIP score is a standard automatic evaluation metric for text-to-image models
- Nearly every topic in this course (diffusion models, latent diffusion, RL from human feedback etc...) touches CLIP in some way

Limitations of CLIP:
- 77-token hard limit on caption, longer prompts are truncated

Variants of the original CLIP work around it: OpenCLIP, SigLIP, FLAVA, ALIGN

Same concept of CLIP can be extended to other modalities

# Autoregressive Generation

Goal of any generative model is to learn the joint distribution over data
in autoregressive models we apply the **chain rule of probability** to obtain an exact lossless decomposition of conditional probabilities
![[Screenshot 2026-03-31 at 1.57.54 PM.png]]
![[Screenshot 2026-03-31 at 1.58.02 PM.png]]
This is an exact identity, no approximation is involved

The recursion becomes explicit when we observe that each factor conditions only all previously generated elements, which grows the joint one variable at a time
Each step requires only the previous context making the computation inherently recursive
An auto-regressive model parameterises each conditional factor with a shared neural network
![[Screenshot 2026-03-31 at 2.02.12 PM.png]]
